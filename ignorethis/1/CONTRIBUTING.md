## ✅ Quick Checklist

Before you open a PR, make sure **every** item below is checked:

- [ ] The resource is **specifically about learning Russian** (not a generic tool with Russian as one of 50 languages)
- [ ] The link **works** — you personally visited it and confirmed the content exists
- [ ] The resource is **not already listed** — search the README for the URL and name
- [ ] The correct **access tag** is applied: `[Free]`, `[Freemium]`, or `[Paid]`
- [ ] The correct **CEFR level** is assigned: A1, A2, B1, B2, C1, C2, or "All"
- [ ] The description is **one concise sentence** ending with a period
- [ ] The entry is placed in the **correct category** (see [Category Guide](#-category-guide))
- [ ] The entry follows the **exact table format** used in that section
- [ ] You added **only one resource** per PR
- [ ] You updated the **resource count** in the Table of Contents and Quick Stats

## 🎯 What We Accept

### Resource Types

We welcome these types of Russian language learning resources:

| Type | Examples |
|------|----------|
| **Courses & Lessons** | MOOCs, structured courses, textbook companions |
| **Reference Material** | Grammar guides, dictionaries, conjugation tables |
| **Practice Tools** | Flashcard decks, exercises, quizzes, worksheets |
| **Media** | Podcasts, YouTube channels, movies, TV shows, music, radio |
| **Reading Material** | Graded readers, parallel texts, literature with annotations |
| **Apps & Software** | Mobile apps, desktop programs, browser extensions |
| **Communities** | Language exchange platforms, forums, Discord servers |
| **NLP & Developer Tools** | GitHub repos, corpora, morphological analyzers, TTS engines |
| **Cultural Context** | History, traditions, holidays, cultural guides that aid language learning |

### Acceptance Tiers

Not every resource qualifies just because it exists. Submissions should meet **at least one** of these criteria:

| Tier | Criteria | Example |
|------|----------|---------|
| **🌟 Gold Standard** | The go-to resource in its category; widely recommended by teachers and learners | RussianForFree.com, MasterRussian, Cooljugator |
| **📈 Rising Star** | Gaining recognition; active development; positive community feedback (500+ users/stars where applicable) | A new app with growing reviews and regular updates |
| **💎 Hidden Gem** | Exceptional quality despite lower visibility — fills a specific gap no other resource covers | A niche academic corpus, a unique verb-aspect visualizer |

**All submissions must**:
- Have content that is **actually accessible** (not behind a broken link, CAPTCHA wall, or geo-block with no workaround)
- Be **actively maintained** or contain timeless content (a static PDF reference is fine; an abandoned web app with broken features is not)
- Provide **genuine educational value** — not just a landing page, ad farm, or link aggregator with no original content

## 🚫 What We Do NOT Accept

The following will be **rejected without further discussion**:

| Rejection Reason | Details |
|-----------------|---------|
| **Not Russian-specific** | A generic tool (e.g., Quizlet, Anki) that merely supports Russian among 50+ languages. The resource must offer something *specifically valuable for Russian learners* — Russian-specific content, Russian-specific features, or Russian-focused curation |
| **Dead links** | The URL returns 404, 500, or a parked domain. Verify before submitting |
| **Paywalled with no free access** | Resources that require payment before seeing ANY content. We accept `[Paid]` tags for premium courses and books, but the landing page must at least describe what you get |
| **AI-generated content farms** | Low-quality sites generated by AI with no editorial oversight, filled with generic content and ads |
| **Link shorteners or redirects** | No bit.ly, tinyurl, or affiliate redirect links. Use the direct URL |
| **Duplicate entries** | The same resource already exists in the list, even under a different name. Search for the URL before submitting |
| **Geo-blocked with no workaround** | Resources that are completely inaccessible outside a specific country with no VPN alternative or mirror |
| **Malware / compromised sites** | Sites serving malware, phishing, or excessive malicious advertising |
| **YouTube playlists** | YouTube playlists are extremely fragile and frequently become unavailable. Submit individual videos or channels instead |
| **Fragmented social media posts** | Individual tweets, TikTok videos, or Instagram posts. Curated collections or channels are acceptable |
| **Resources requiring login to view** | If the core content cannot be previewed without creating an account, the resource does not belong here. Registration walls for *saving progress* are acceptable; registration walls for *viewing content* are not |

## 📝 Entry Format

### Standard Table Format

Most sections use this table format:

```markdown
| # | Resource | Description | Level |
|:---:|----------|-------------|:-----:|
| 1 | [Resource Name](https://example.com) | Concise description of what this resource offers [Free] | A1 |
```

### Key Formatting Rules

1. **Link text** = the official name of the resource (not a description)
2. **URL** = the most direct link to the resource (not a homepage if a subpage is more useful)
3. **Description** = one sentence, ending with a period, stating what the resource IS and DOES
4. **Access tag** = `[Free]`, `[Freemium]`, or `[Paid]` — placed at the END of the description, inside the same cell
5. **Level** = CEFR level (A1, A2, B1, B2, C1, C2) or `All` if it spans multiple levels

### ✅ Good vs ❌ Bad Examples

#### Description

| ✅ Good | ❌ Bad | Why |
|---------|--------|-----|
| `Complete conjugation tables for all Russian verbs with stress marks [Free]` | `Great website!!! Check it out 🎉` | Good: factual, specific, ends with period |
| `Interactive declension drills for all six cases with audio [Freemium]` | `A tool for declining Russian words` | Good: mentions specific features + correct tag |
| `University-level grammar reference from Cornell with exercises [Free]` | `Cornell Russian grammar - this is a really awesome and comprehensive resource that you should definitely check out if you want to learn Russian grammar properly` | Good: concise; Bad: promotional, too long |
| `AI-powered text-to-speech generation with natural Russian voices [Freemium]` | `Russian TTS` | Good: descriptive; Bad: too vague |

#### Access Tag Placement

| ✅ Good | ❌ Bad |
|---------|--------|
| `Free grammar lessons with audio for every lesson [Free]` | `Free grammar lessons with audio` |
| `Structured course with grammar explanations and review [Freemium]` | `Structured course (Freemium) with grammar explanations` |
| `University-standard Russian textbook with audio [Paid]` | `University-standard Russian textbook with audio (Paid)` |

### Special Prefixes

Use these prefixes **before** the link when applicable:

| Prefix | When to Use | Example |
|--------|-------------|---------|
| `📄` | Downloadable PDF resource | `📄 [Russian Grammar Chart (PDF)](url)` |
| `▶️` | Video content (YouTube, Vimeo, etc.) | `▶️ [Complete Russian Course 9 Hours](url)` |
| `🎙️` | Podcast / audio-only content | `🎙️ [Speaking Russian Podcast](url)` |

### Warnings

If a resource has a known accessibility issue that does not warrant deletion (e.g., occasional CAPTCHA, HTTP-only), append a warning **after** the table row:

```markdown
| 6 | [RussianLessons — Verbs of Motion](url) | Description [Free] | All | ⚠️ *Site often shows CAPTCHA*
```

Warning types:
- `⚠️ *Site often shows CAPTCHA*` — Site triggers CAPTCHA/robot challenges frequently
- `⚠️ *Site often shows Cloudflare challenge*` — Cloudflare-protected with aggressive bot detection
- `⚠️ *HTTP-only site (no HTTPS)*` — Site does not use HTTPS encryption
- `⚠️ *Site frequently unreachable*` — Intermittent downtime
- `⚠️ *Email required for PDF download*` — Free but requires email to access
- `⚠️ *Site often shows CAPTCHA/Robot Challenge*` — Aggressive anti-bot that may block legitimate users
- `⚠️ *Homepage compromised; only this category page works*` — Part of the site is compromised/inaccessible

> **Important**: Warnings are for resources that still provide value despite issues. If a site is completely inaccessible, dead, or compromised in a way that makes it unusable, it should be **removed entirely**, not warned.

## 🏷️ Tagging Rules

### Access Tags (REQUIRED — every entry must have one)

| Tag | Definition | Examples |
|-----|-----------|----------|
| `[Free]` | The core content is **fully accessible at no cost** with no time limit, no credit card required, and no feature-gating on essential functionality. Free registration is acceptable. | RussianForFree.com, MasterRussian, Wikipedia articles, open-source GitHub repos |
| `[Freemium]` | There is a **meaningful free tier** that provides genuine value, but premium features, advanced content, or higher usage limits require payment. The free tier must be usable for actual learning — not just a 7-day trial or teaser. | Duolingo (free with ads + premium tier), Lingvist (free daily limit), Busuu (free basic + premium) |
| `[Paid]` | The resource **requires payment** to access the core content. This includes textbooks, subscription-only courses, and services with no usable free tier. Only use this for high-quality resources that fill a gap no free alternative covers. | Pimsleur, Amazon textbooks, specialized certification prep courses |

#### Tagging Decision Flowchart

```
Does the resource cost money to access core content?
├── NO → Is there a free tier that provides genuine learning value?
│   ├── YES, full access for free → [Free]
│   └── YES, but limited (daily caps, feature gates, ads) → [Freemium]
└── YES → Is it a high-quality resource with no free alternative?
    ├── YES → [Paid]
    └── NO → Do not add it
```

**Common mistakes**:
- ❌ Tagging a cloud API service (AWS, Google Cloud, Azure) as `[Free]` — these are `[Freemium]` at best
- ❌ Tagging an app with a "7-day free trial" as `[Freemium]` — if the free tier expires, it is `[Paid]`
- ❌ Tagging a resource as `[Free]` when registration requires a credit card — that is `[Freemium]` or `[Paid]`
- ❌ Omitting the tag entirely — every entry MUST have one

### CEFR Level Tags (REQUIRED)

Assign the appropriate CEFR level based on the resource's target audience:

| Tag | Meaning | Typical Content |
|-----|---------|-----------------|
| `A1` | Beginner | Alphabet, basic phrases, first 100 words |
| `A2` | Elementary | Simple conversations, basic grammar, everyday vocabulary |
| `B1` | Intermediate | Complex sentences, reading short articles, expressing opinions |
| `B2` | Upper Intermediate | News articles, podcasts at normal speed, essay writing |
| `C1` | Advanced | Academic texts, professional communication, nuanced grammar |
| `C2` | Proficiency | Native-level literature, specialized vocabulary, cultural nuance |
| `All` | Multiple / All levels | Reference tools, dictionaries, resources spanning A1–C2 |

**When in doubt**: Use `All` for reference materials and tools that are useful at any level (dictionaries, conjugators, keyboard tools). Use the specific level for targeted learning content (A1 courses, C2 listening practice).

## 🚀 Submitting a Pull Request

### Step-by-Step Process

1. **Fork** the repository
2. **Create a branch**: `git checkout -b add/resource-name`
3. **Add your resource** to the appropriate section in `README.md`
4. **Verify the link works** — visit it yourself in a browser
5. **Update the Table of Contents** — increment the count for the category you added to (e.g., `*(11)*` → `*(12)*`)
6. **Update Quick Stats** — increment the "Total Resources" count in the stats table at the top
7. **Update Resource Statistics** — update the counts in the bottom statistics table if present
8. **Commit** with a descriptive message: `git commit -m 'Add [Resource Name] to [Category]'`
9. **Open a Pull Request** using the template below

### One Resource Per PR

Each PR should add, remove, or modify **only one resource**. This makes review faster and allows partial acceptance. If you want to add 5 resources, open 5 separate PRs.

### PR Title Format

```
Add [Resource Name] to [Category Name]
```

Examples:
- ✅ `Add RussianWithDasha to YouTube & Video`
- ✅ `Add OpenCorpora Dataset to GitHub Repositories & NLP`
- ❌ `Added some stuff`
- ❌ `Updated README`
- ❌ `New resource`

### PR Template

Copy and fill in this template when opening your PR:

```markdown
## Resource Being Added

**Name**: [Resource Name]
**URL**: [https://...]
**Category**: [Category Name from the list]
**Access Tag**: [Free / Freemium / Paid]
**CEFR Level**: [A1 / A2 / B1 / B2 / C1 / C2 / All]

## Checklist

- [ ] I have personally visited the URL and confirmed the content exists and is accessible
- [ ] The resource is specifically about learning or processing Russian
- [ ] I searched the README and this resource is not already listed
- [ ] The access tag accurately reflects the pricing model ([Free] = fully free, [Freemium] = meaningful free tier + paid, [Paid] = payment required)
- [ ] The description is one concise sentence ending with a period
- [ ] The entry follows the exact table format used in that section
- [ ] I added only one resource in this PR
- [ ] I updated the Table of Contents count for the affected category
- [ ] I updated the Total Resources count in Quick Stats
- [ ] This PR is not AI-generated spam

## Description of the Resource

[1-3 sentences explaining what this resource offers and why it deserves to be on the list. This helps reviewers understand the value.]
```

## 🐛 Reporting Issues

### Broken Links

If you find a broken link, please **open an issue** with:

```markdown
**Broken Link Report**

- **Resource Name**: [Name]
- **Category**: [Category]
- **URL**: [the broken URL]
- **What happened**: [404 / Parked domain / Redirects elsewhere / CAPTCHA wall / Other]
- **Last known working** (if you know): [date or "unknown"]
- **Suggested action**: [Remove / Replace with archive link / Update URL / Tag with warning]
```

### Incorrect Information

If a resource has an incorrect tag, description, or CEFR level, open an issue with:

```markdown
**Correction Report**

- **Resource Name**: [Name]
- **Current entry**: [copy the current line from README]
- **What is wrong**: [Tag should be Freemium not Free / Description is inaccurate / Level is wrong]
- **Corrected entry**: [proposed corrected line]
- **Evidence**: [URL showing pricing page / documentation showing correct level / etc.]
```

### Suggesting a New Category

If you think a new category is needed:

```markdown
**New Category Proposal**

- **Proposed name**: [Category Name]
- **Emoji**: [suggested emoji]
- **Description**: [what resources would go here]
- **Example resources**: [at least 3 resources that would belong in this category]
- **Why existing categories don't fit**: [explain why these resources don't belong in current categories]
```

> New categories require a minimum of **3 resources** to be created. Do not create categories with only 1-2 items.

## 🗑️ Removing a Resource

Resources are removed when they:

- **No longer exist** (404, parked domain, expired)
- **Are no longer maintained** and have broken features (not just "old" — timeless content is fine)
- **Have been replaced** by a clearly superior resource (add the replacement first)
- **Violate our guidelines** (e.g., was discovered to be malware, became a paywall with no free access)
- **Are not Russian-specific** (e.g., a generic tool that merely "supports" Russian)

### Removal Process

1. **Open a PR** that removes the entry line from the table
2. **Update counts** in the Table of Contents and Resource Statistics
3. **In the PR description**, explain:
   - Why the resource is being removed
   - When you last verified the issue
   - Whether an alternative resource should replace it

### When to Add a Warning Instead of Removing

If a resource is **still valuable** but has accessibility issues, add a `⚠️` warning rather than removing it:

| Situation | Action |
|-----------|--------|
| Site shows CAPTCHA on first visit but content loads after solving | Add `⚠️ *Site often shows CAPTCHA*` |
| Site is HTTP-only (no HTTPS) but works fine | Add `⚠️ *HTTP-only site (no HTTPS)*` |
| Site is intermittently down (works most of the time) | Add `⚠️ *Site frequently unreachable*` |
| Site requires email for a free PDF download | Add `⚠️ *Email required for PDF download*` |
| Homepage is compromised but a subpage still works | Add `⚠️ *Homepage compromised; only this category page works*` |
| Site returns 404 completely | **Remove entirely** — do not warn |
| Site is a malware/phishing risk | **Remove entirely** — do not warn |
| YouTube playlist is unavailable | **Remove entirely** — these never come back |

## 🔧 Curation Policy & Maintenance

### How We Keep the List Fresh

This list is **actively curated**, not just accumulated. Maintenance includes:

- **Periodic link verification** — URLs are visited and checked for accessibility
- **Tag audits** — Resources tagged `[Free]` are re-verified; those that added paywalls are changed to `[Freemium]` or `[Paid]`
- **Description accuracy** — Descriptions are updated when resources change their features
- **Pruning** — Dead, outdated, and no-longer-maintained resources are removed
- **Warning updates** — Warnings are added or removed as site conditions change

### What Gets Pruned

| Priority for Removal | Examples |
|---------------------|----------|
| **Dead links** | 404s, parked domains, DNS failures |
| **YouTube playlists** | These are extremely fragile — most become unavailable within 1-2 years |
| **Compromised sites** | Sites serving malware, phishing, or redirecting to suspicious domains |
| **Non-Russian resources** | Generic tools with no Russian-specific value added |
| **Mislabeled resources** | Resources tagged `[Free]` that are actually `[Freemium]` or `[Paid]` after verification |
| **Abandoned projects** | GitHub repos archived or with no commits for 3+ years AND open critical issues |
| **Low-value duplicates** | Resources that offer nothing beyond what existing entries already provide |

### What Gets Kept

| Keep Even If... | Reason |
|-----------------|--------|
| **Old but timeless** | A well-written grammar PDF from 2010 is still valuable |
| **Has warnings** | Sites with CAPTCHA issues that still provide unique content |
| **Paid but unique** | High-quality paid resources with no free alternative |
| **Niche but useful** | Specialized resources (e.g., medical Russian, legal Russian) with small audiences |

## 📐 Style & Formatting Reference

### Table Structure

All resource sections use this consistent table format:

```markdown
| # | Resource | Description | Level |
|:---:|----------|-------------|:-----:|
| 1 | [Name](URL) | Description text [Tag] | A1 |
| 2 | ▶️ [Name](URL) | Description text [Tag] | All |
| 3 | 📄 [Name (PDF)](URL) | Description text [Tag] | B1 |
```

- Column `#` is centered and contains the row number
- Column `Resource` is left-aligned with the link (and optional prefix emoji)
- Column `Description` is left-aligned, contains the description + access tag
- Column `Level` is centered with the CEFR level

### Alphabetical Ordering

Within each category, resources are **NOT** strictly alphabetical (they are ordered by relevance and type). However:
- **Do not** reorder existing entries unless doing a full section cleanup
- **Add new entries at the end** of the appropriate sub-section
- When doing a full cleanup, alphabetical order within sub-sections is preferred

### Numbering

Row numbers should be sequential within each sub-section. When adding a new entry:
- Use the next available number in that sub-section
- When removing an entry, **do not** renumber the remaining entries (leave gaps — this helps with referencing)

### Markdown Conventions

- Use **GitHub Flavored Markdown** (GFM)
- No trailing whitespace
- No URL shorteners
- Section dividers use: `--- 🇷🇺 --- 🇷🇺 --- 🇷🇺 ---`
- Collapsible sections use `<details><summary>` for categories with 30+ items
- Emoji usage is consistent per section (see category headers)

### Count Synchronization

When adding or removing a resource, you MUST update **all three** of these:

1. **Table of Contents** — The number in parentheses next to the category name (e.g., `*(11)*`)
2. **Quick Stats table** — The "Total Resources" count
3. **Resource Statistics table** — The per-category count in the bottom statistics section

Failing to update counts is the most common PR mistake and will cause your PR to be delayed.
